About the Project

Project Zomboolean was a 4-week-long project aimed at young teens and created using the Unity engine. It's an interactive top-down shooting game where your choices matter—from equipping different weapons, upgrading your skills, and buying more land to slay all the evil zombies.

Teamwork

This project was not possible without the contribution of all team members involved.

  • To communicate and plan our work, we used a combination of written and vocal stand-ups. We had an end-of-day meeting every day that refined our vision and set clear short- and long-term goals for the project. During the project month, we gave each other feedback and set improvement plans for the group as a whole.

  • We used SCRUM intensively, using Trello, meetings, and revisions each week based on the SCRUM method of teamwork.

  • We also used GitHub to collaborate and seamlessly progress from brainstorming to designing and finally into production. It helped us keep track of all changes and backtrack to solve the challenges we faced!

My Contribution

  • Design Phase

    I worked on the TGDD, which set the foundation for our theme, genre, game mechanics, and partly the UI design. I researched the plane purchasing mechanic and presented it to my group.

  • UI/UX

    I created animated damage numbers using a mix of scripting and animation events to create the tempo and loop of the animation.

    I also created and implemented the Plane Cost UI, indexing the object into the plane allowed me to perfect the positioning and quality of the UI element.

    I created the gold top-bar UI, implementing it in a way where numbers get parsed into suffixes—for example, 1,000,000 becomes 1M.

  • Programming/Coding

    Player Camera: I programmed the main camera to always follow the player at a certain angle and created an angle locker so the UI elements of damage and plane cost always face the camera.

    I created randomized rewards for killing zombies, making them public for easier accessibility. I also randomized which animation plays for the damage pop-up before instantiating the animation.

    I also created the plane purchasing mechanic, looping through all the children of the plane components to set/update the pricing for all buy signs and updating the UI Text element with the correct amount.

    Player Movement: I implemented the player movement using a dynamic joystick package for the control UI, utilizing engine physics to build the player controls for smooth movement.
